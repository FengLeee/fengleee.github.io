    英文验证码的url：'https://www.zhihu.com/captcha.gif?r=1501839818639&type=login'和目前知乎用的
url："https://www.zhihu.com/captcha.gif?r=1501839818639&type=login&lang=cn"是不一样的．我们去掉了lang=cn字段
这样返回给我们的验证码就是普通的英文验证码，修改验证码类型后，在form表单中也要更改相应的验证码类型，'captcha_type': 'cn',
改为'captcha_type': 'en',

captcha_time = str(time.time()).split('.')[0]+str(time.clock()).split('.')[1][:3]
captcha_url = 'https://www.zhihu.com/captcha.gif?r={0}&type=login'.format(captcha_time)
form_data = {
        '_xsrf': get_xsrf(),
        'password': passwd,
        'captcha': get_captcha(),
        'captcha_type': 'en',
        'email': account
    }

    这样我们post form_data字段就能成功登陆了

scrapy中间件

spiderMiddleware

spiderMiddleware在spider和engine中间,scrapy默认的spiderMiddleware module有httperror,URLlength,referer,offsite等,我们自定义时可以参考实现自己的middleware,以下是几个必须实现的方法

process_spider_output(self, response, spider)  # spider送给engine时调用
process_spider_put(self, response, spider)    # engine返回spider时调用
process_spider_exception(self, response, exception, spider): #当爬出异常时调用

Downloadermiddleware

Downloadermiddleware在engine和download中间,scrapy默认的Downloadermiddleware module有httpproxy,redirect,retry,useragent等,我们自定义时可以参考实现自己的middleware,以下是几个必须实现的方法

process_request(self, request, spider):
process_response(self, request, response, spider):
process_exception(self, request, exception, spider):

这些方法和spidermiddleware方法的作用相同

简明Python教程

发表于 2017-04-04 | 分类于 读书

局部变量

当你在一个函数的定义中声明变量时，它们不会以任何方式与身处函数之外但具有相同名称
的变量产生关系，也就是说，这些变量名只存在于函数这一局部（Local）。这被称为变量的
作用域（Scope）。所有变量的作用域是它们被定义的块，从定义它们的名字的定义点开始。
案例

x = 50

def func(x):
    print('x is', x)
    x = 2
    print('Changed local x to', x)
func(x)

print('x is still', x)

输出

$ python function_local.py
x is 50
Changed local x to 2
x is still 50

可变参数

当我们声明一个诸如 *param 的星号参数时，从此处开始直到结束的所有位置参数

（Positional Arguments）都将被收集并汇集成一个称为“param”的元组（Tuple）。

类似地，当我们声明一个诸如 **param 的双星号参数时，从此处开始直至结束的所有关键字

参数都将被收集并汇集成一个名为 param 的字典（Dictionary）。(传参时位置参数要在关键字参数前面)

格式化输出

a = 10
print ('a= %s' %(a))
print ('a= {}'.format(a))
print ('a=', a)

帮助方法

help(list)

list.__doc__

dir(list)

序列

Python 中有四种内置的数据结构——列表（List）、元组（Tuple）、字典（Dictionary）和集合（Set）

列表、元组和字符串可以看作序列（Sequence）的某种表现形式，可是究竟什么是序列，它
又有什么特别之处？

序列的主要功能是资格测试（Membership Test）（也就是 in 与 not in 表达式）和索引
操作（Indexing Operations），它们能够允许我们直接获取序列中的特定项目。
上面所提到的序列的三种形态——列表、元组与字符串，同样拥有一种切片（Slicing）运算
符，它能够允许我们序列中的某段切片——也就是序列之中的一部分。

集合

集合（Set）是简单对象的无序集合（Collection）。当集合中的项目存在与否比起次序或其出
现次数更加重要时，我们就会使用集合。

MYSQL配置utf8mb4

发表于 2017-04-04 | 分类于 MySQL

对于Python来说 最好用pymsql. MySQLDB在win10环境下charset不能设置为utf8mb4这是一个很大的坑

在mysql 5.5 之前，UTF-8编码只支持1-3个字节；从MySQL 5.5开始，可以支持4个字节UTF编码 utf8mb4 ，一个字符能够支持更多的字符集，也能够支持更多表情符号。

一：首先我们修改my.cnf参数

[client]
# 客户端连接
default-character-set=utf8mb4


[mysql]
# 远程连接
default-character-set=utf8mb4


[mysqld]
# 在window下不能直接添加 要找到默认的character-set-server 然后覆盖
character-set-server = utf8mb4
collation-server = utf8mb4_unicode_ci
init_connect = 'SET NAMES utf8mb4'
character-set-client-handshake = true

登录数据库检查是否如下：

Item Loaders 提供了一个便利的机制来帮助 populating(填充) scrapted Items；虽然，Items 可以通过它类似 dict API 来填充，Item
Loaders 提供了更多便利的方法来进行 populates；
简而言之，Items 提供了被爬取数据的一个容器，而 Item Loaders 为该容器提供了 populating 的机制；
Item Loaders 的目的是为不同的 field 设计出更为高效、简单的可覆盖和可扩展的解析规则，可以用于不同的 spiders，以及不同的 source format (比如 HTML，XML 等等)，而不至于是维护编程噩梦；
看一个例子

item_loader = MyItemLoader(item=ArticleSpiderItem(), response=response)
item_loader.add_xpath('title', "//div[@class='entry-header']/h1/text()")
item_loader.add_xpath('create_date', "//div[@class ='entry-meta']/p/text()")
item_loader.add_value('url', response.url)
item = item_loader.load_item()

最后，当所有的数据都已经收集好了以后，将会通过 [ItemLoader.load_item()] 将其转换为 Item 并返回；

Selenium之集成到Scrapy中

发表于 2016-11-24 | 分类于 scrapy

通过前面的学习了解了Selenium的具体使用,这一次我们把Selenium集成到Scrapy中

第一种方式:

直接把Selenium集成到middleware中的process_request()方法中,但是这样每次都会重新生成一个webdriver对象,效率不高

第二种方式

在middleware类中进行初始化,这个可以保证只存在一个webdriver对象,不会重复new对象,但是这种方式有个缺点,当爬虫关闭时 webdriver不会自动关闭,所以我们引申出第三种方式

第三种方式

在第三种方式中,我们把selenium和我们的spider绑定在一起,代码如下,在spider进行初始化时,生成webdriver对象,通过spider传递到middleware.直接调用.同时我们也绑定了一个函数spider_closed(),当Scrapy通过signals发出spider_closed信号时调用spider_closed函数关闭webdriver

def __init__(self):
    # 让webdriver不加载图片
    chromeOptions = webdriver.ChromeOptions()
    prefs = {"profile.managed_default_content_settings.images": 2}
    chromeOptions.add_experimental_option('prefs', prefs)
    self.browser = webdriver.Chrome('/home/python/Desktop/soft/chromedriver', chrome_options=chromeOptions)
    # spider_closed() 当spider_closed信号发出时执行
    dispatcher.connect(self.spider_closed, signals.spider_closed)
    super(ArticleboleSpider, self).__init__()

def spider_closed(self):
    print 'browser 退出了'
    self.browser.quit()

Selenium之三种等待方式

发表于 2016-11-04 | 分类于 selenium

1. 强制等待

# -*- coding: utf-8 -*-
from selenium import webdriver
from time import sleep

driver = webdriver.Firefox()
driver.get('https://fengleee.github.io')

sleep(3)  # 强制等待3秒再执行下一步

print driver.current_url
driver.quit()

这种叫强制等待，不管你浏览器是否加载完了，程序都得等待3秒，3秒一到，继续执行下面的代码，作为调试很有用，有时候也可以在代码里这样等待，不过不建议总用这种等待方式，太死板，严重影响程序执行速度。
